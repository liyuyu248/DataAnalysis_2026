# DataAnalysis_2026

This repository is organized **by module**, following the course requirement that each module should have its own folder.

## Current status
At present, only **Module 2: Data Manipulation and Cleaning** is fully completed, because the uploaded materials include only:

- `Practice 1.pdf`
- `Data_Manipulation_Process.R`
- `DataSet_No_Details.csv`

So I completed the module that can be supported by the provided files, and created placeholders for additional modules.

## Repository structure

```text
DataAnalysis_2026/
├── Module_1_Business_Case_Definition/
├── Module_2_Data_Manipulation_and_Cleaning/
├── Module_3_Data_Visualization/
└── Module_04_Statistical_Analysis/
```

## What each module folder should contain

Each module should contain:

- `README.md` — assignment description, dataset description, software version, procedures used, conclusions
- `data/` — source data files
- `code/` — scripts / notebooks
- `output/` — generated graphs, tables, cleaned data, model results

## Completed module

`Module_2_Data_Manipulation_and_Cleaning` already contains:

- source data
- original teacher script
- completed R script
- result tables
- plots

## How to use this repository

1. Keep the completed Module 2 folder as-is.
2. Rename `Module_1_Your_Module_Name` and `Module_3_Your_Module_Name` to your real module names.
3. For additional modules, copy `Module_Template/` and rename it.
4. Put each module's files into `data/`, `code/`, and `output/`.
5. Commit and upload the whole repository to GitHub as a **public** repository.
